01

Kurulum

Sadece iki dosya: glint-input.css ve glint-input.js. CSS'i <head>'e, JS'i sayfa sonuna ya da <head>'e koy. Harici bağımlılık yoktur. Tüm bileşenler (input, checkbox, kontroller, select, alanlar, slider, upload, color, picker) tek pakettedir.

Tembel (lazy) işleme. Her bileşen yalnızca sayfada o bileşene ait bir element varsa başlatılır. Kullanmadığın bileşen ne örnek oluşturur ne listener bağlar — tek dosya olmasına rağmen sistemi yormaz. Kullanılmayan CSS kuralları da atıldır.
<head>
  <link rel="stylesheet" href="glint-input.css">
</head>
<body>
  ...

  <script src="glint-input.js"></script>
</body>

Hepsi bu. Glint, sayfa yüklenince .glint-input-group, .glint-checkbox ve diğer tüm bileşen elemanlarını otomatik keşfedip başlatır. Bir MutationObserver sayesinde sonradan eklenen alanlar (AJAX, SPA, dinamik form) da kendiliğinden devreye girer.

Tema rengi. Tüm bileşenler vurgu rengini --bs-primary'den okur (yoksa emerald #059669). Bootstrap kullanıyorsan otomatik uyumlu; kullanmıyorsan :root { --bs-primary: #2563eb; } ile kendi rengini ver. Koyu mod için kök elemana data-bs-theme="dark" ekle (bu sayfanın sağ üstündeki düğme de bunu yapıyor).

03 · Bileşen

Glint Input

Metin alanları için animasyonlu bir katman: odakta SVG ile çizilen kenarlık, yüzen etiket, her tuşta karakter animasyonu (giriş, yapıştırma dalgası, silme hayaleti), hata durumu, çok satırlı mod, koyu mod ve erişilebilirlik.

Temel metin alanı

Etiket, odakta veya değer girildiğinde yukarı süzülür ve kenarlığı temiz keser.

Canlı önizleme
<div class="glint-input-group">
  <input class="glint-input" id="email" type="email">
  <label class="glint-label" for="email">E-posta adresi</label>
</div>
<div class="glint-input-group">
  <input asp-for="Email" class="glint-input" />
  <label asp-for="Email" class="glint-label"></label>
</div>
Yüzen etiket, üstünde durduğu yüzeyin arka planını background: inherit ile miras alır. İç içe / şeffaf katmanlarda netlik için yüzey rengini ver: .glint-input-group { --glint-label-bg: #fff; }

Şifre alanı, göster / gizle

type="password" alanlarında karakterler nokta olarak maskelenir. Sağdaki aksiyon butonu için .glint-field__action kullan; Glint butonu konumlandırır, sağ boşluğu açar ve tıklamada odağın alandan kaymasını engeller (animasyon flaşlamaz).

Canlı önizleme
<div class="glint-input-group">
  <input class="glint-input" id="pw" type="password">
  <label class="glint-label" for="pw">Şifre</label>
  <button type="button" class="glint-field__action" id="pwTg"
          aria-label="Şifreyi göster">
    <svg>...göz ikonu...</svg>
  </button>
</div>
const pw = document.getElementById('pw');
document.getElementById('pwTg').addEventListener('click', () => {
  pw.type = pw.type === 'password' ? 'text' : 'password';

  // Overlay maskesini güncelle (isPassword örnekte cache'lenir)
  const inst = Glint.Input._getInstance(pw);
  if (inst) { inst.isPassword = pw.type === 'password'; inst.syncOverlay(false); }
});

Temizle butonu (değer varken görünür)

Butona .glint-field__action--clear eklersen alan boşken gizlenir, değer girilince yumuşakça belirir. Değeri animasyonlu silmek için önce beforeinput, sonra input olayını tetikle.

Canlı önizleme · yaz ve sil
<div class="glint-input-group">
  <input class="glint-input" id="search">
  <label class="glint-label" for="search">Arama</label>
  <button type="button"
          class="glint-field__action glint-field__action--clear"
          id="clrBtn" aria-label="Temizle">
    <svg>...çarpı ikonu...</svg>
  </button>
</div>
const search = document.getElementById('search');
document.getElementById('clrBtn').addEventListener('click', () => {
  search.dispatchEvent(new Event('beforeinput', { bubbles: true, cancelable: true }));
  search.value = '';
  search.dispatchEvent(new Event('input', { bubbles: true })); // overlay'i günceller
  search.focus();
});

Çok satırlı (textarea)

Bir <textarea> kullan; Glint otomatik algılayıp .glint-input-group--multiline ekler. Varsayılan auto-grow'dur (içerik büyüdükçe uzar, tavana kadar). Sabit yükseklik için gruba .glint-input-group--fixed ekle.

Canlı önizleme · auto-grow
<!-- Auto-grow (varsayılan) -->
<div class="glint-input-group">
  <textarea class="glint-input" id="desc"></textarea>
  <label class="glint-label" for="desc">Açıklama</label>
</div>

<!-- Sabit yükseklik -->
<div class="glint-input-group glint-input-group--fixed">
  <textarea class="glint-input" rows="5"></textarea>
  <label class="glint-label">Notlar</label>
</div>
<div class="glint-input-group">
  <textarea asp-for="Description" class="glint-input"></textarea>
  <label asp-for="Description" class="glint-label"></label>
</div>

Satır sayısı: --glint-textarea-min-rows, --glint-textarea-max-rows. Satır kırılması değişince mevcut karakterler yeni konumlarına yumuşakça kayar (FLIP).

Hata durumu

Glint.Input.setError(el, "mesaj") kenarlığı kırmızıya çevirir, alanı titretir ve mesajı alanın altında gösterir (içerik aşağı kayar, yer açılır). Kullanıcı yazmaya başlayınca hata otomatik temizlenir.

Canlı önizleme
const el = document.getElementById('username');

// Hata göster
Glint.Input.setError(el, 'Bu kullanıcı adı zaten alınmış.');

// Hata temizle (kullanıcı yazınca da otomatik temizlenir)
Glint.Input.clearError(el);

MVC doğrulamasını (jQuery unobtrusive) bu API'ye bağlamak için ASP.NET Core MVC bölümündeki köprüye bak.

Ekstra otomatik davranışlar

  • Toplu silme: Ctrl+A ile tüm metni seçip Backspace/Delete'e basınca karakterler sağdan sola animasyonlu silinir.
  • Autofill: tarayıcı otomatik doldurması algılanır; etiket yukarı süzülür, overlay senkronlanır.
  • IME: birleşik karakter girişi (CJK vb.) sırasında overlay doğru senkronlanır.
  • Kaydırma: uzun metinde overlay, native imleç ve scroll ile hizalı kalır.

Otomatik şifre göster / gizle

type="password" alanlarına Glint, modern çizgisel (stroke) bir göz / göz-çizgili ikonu kendiliğinden ekler — ekstra buton ya da JS yazmana gerek yok. İkon, alanın maskesini (• ↔ gerçek karakter) tek tıkla çevirir. Devre dışı bırakmak için gruba data-glint-no-toggle ekle.

Canlı önizleme
<!-- İkon otomatik eklenir; hiçbir şey yapmana gerek yok -->
<div class="glint-input-group">
  <input class="glint-input" id="pw" type="password">
  <label class="glint-label" for="pw">Şifre</label>
</div>

<!-- Opt-out: ikon istemiyorsan -->
<div class="glint-input-group" data-glint-no-toggle>
  <input class="glint-input" id="pw2" type="password">
  <label class="glint-label" for="pw2">Şifre</label>
</div>
<div class="glint-input-group">
  <input asp-for="Password" type="password" class="glint-input" />
  <label asp-for="Password" class="glint-label"></label>
</div>
İkonun durumu aria-pressed ile erişilebilir biçimde duyurulur; göster moduna geçince butona .is-revealed sınıfı eklenir (CSS göz ↔ göz-çizgili ikon takasını yapar).

Parola güç göstergesi

Gruba data-glint-strength ekle; alanın altına 4 segmentli bir güç çubuğu gelir (zayıf → güçlü). Skor, uzunluk + büyük/küçük harf karışımı + rakam & sembol kriterleriyle hesaplanır ve ekran okuyuculara aria-live ile duyurulur. Yalnızca type="password" alanlarında çalışır.

Canlı önizleme
<div class="glint-input-group" data-glint-strength>
  <input class="glint-input" id="pw" type="password">
  <label class="glint-label" for="pw">Yeni şifre</label>
</div>
<div class="glint-input-group" data-glint-strength>
  <input asp-for="NewPassword" type="password" class="glint-input" />
  <label asp-for="NewPassword" class="glint-label"></label>
</div>
Çubuk renkleri durum token'larından gelir: --glint-state-error (1), --glint-state-warning (2), --glint-state-info (3), --glint-state-success (4). Güç metni görsel olarak gizli, erişilebilirlik için canlıdır.

Karakter sayacı

Gruba data-glint-counter ekle; alanın sağ-altında canlı karakter sayısı belirir. Inputta maxlength varsa sayaç otomatik açılır ve 12 / 50 biçiminde gösterir; sınıra yaklaşınca --warn, dolduğunda --full rengine geçer.

Canlı önizleme
<!-- maxlength varsa sayaç kendiliğinden gelir (12 / 50) -->
<div class="glint-input-group">
  <input class="glint-input" id="title" maxlength="50">
  <label class="glint-label" for="title">Başlık</label>
</div>

<!-- Sınır yok, yalnız sayım istersen -->
<div class="glint-input-group" data-glint-counter>
  <input class="glint-input" id="note">
  <label class="glint-label" for="note">Not</label>
</div>
<div class="glint-input-group">
  <input asp-for="Title" class="glint-input" maxlength="50" />
  <label asp-for="Title" class="glint-label"></label>
</div>

Temizle (×) butonu

Gruba data-glint-clearable ekle; sağda bir temizle (×) butonu çıkar. Buton yalnızca alanda değer varken görünür (boşken ölçeklenip kaybolur), tıklanınca değeri siler, odağı alanda tutar ve input olayını tetikler.

Canlı önizleme
<div class="glint-input-group" data-glint-clearable>
  <input class="glint-input" id="q" type="search">
  <label class="glint-label" for="q">Ara</label>
</div>
<div class="glint-input-group" data-glint-clearable>
  <input asp-for="Query" type="search" class="glint-input" />
  <label asp-for="Query" class="glint-label"></label>
</div>
Görünürlük tamamen CSS ile yönetilir: .glint-input-group:not(.glint-has-value) .glint-field__action--clear butonu gizler. Şifre toggle'ı ile birlikte kullanırsan ikinci aksiyon otomatik sola kayar ve sağ boşluk genişler.

Binlik ayraç

Gruba data-glint-group-thousands ekle; yazdıkça rakamlar binlik gruplarına ayrılır (1.234.567). Biçimleme Glint.format ile yapılır, imleç konumu korunur. Ham (yalın) değer her zaman input'un data-glint-raw attribute'unda tutulur — POST etmek için gizli bir alana yansıt.

Canlı önizleme
<div class="glint-input-group" data-glint-group-thousands>
  <input class="glint-input" id="amount" inputmode="numeric">
  <label class="glint-label" for="amount">Tutar</label>
</div>

<!-- Ham değeri POST etmek için: data-glint-raw'ı gizli alana yansıt -->
<input type="hidden" name="Amount" id="amount__raw">
<div class="glint-input-group" data-glint-group-thousands>
  <input class="glint-input" id="amount" inputmode="numeric" />
  <label class="glint-label" for="amount">Tutar</label>
</div>
<input asp-for="Amount" type="hidden" id="amount__raw" />
// Görsel alan binlik ayraçlı; ham değeri (yalın rakam) gizli alana taşı
const fld = document.getElementById('amount');
const raw = document.getElementById('amount__raw');
const sync = () => { raw.value = fld.getAttribute('data-glint-raw') || ''; };
fld.addEventListener('input', sync); sync();
Glint, yalın değeri yalnızca data-glint-raw attribute'una yazar — otomatik gizli alan üretmez. Server'a yalın değeri göndermek için yukarıdaki gibi küçük bir köprü kullan (ya da formu gönderirken data-glint-raw'ı oku).

</div>

<!-- Enter tuşu "Ara" ipucu için -->
<div class="glint-input-group" data-glint-search>
  <input class="glint-input" id="q" type="search">
  <label class="glint-label" for="q">Ara</label>
</div>
<div class="glint-input-group">

Güç göstergesi · tüm skor seviyeleri

Aynı data-glint-strength davranışı, dört farklı parolayla yan yana: skor uzunluk + harf karışımı + rakam & sembol kriterleriyle 1→4 arasında belirlenir ve çubuk durum token'larına göre renklenir. Bir alana yazarak skorun canlı değiştiğini görebilirsin — her seviye ayrı bir alanda dondurulmuştur.

Canlı önizleme
<!-- Tek attribute; skor değere göre 1..4 arası canlı hesaplanır -->
<div class="glint-input-group" data-glint-strength>
  <input class="glint-input" id="pw" type="password">
  <label class="glint-label" for="pw">Yeni şifre</label>
</div>
Skor mantığı: 8+ karakter +1, 12+ karakter +1, küçük ve büyük harf birlikte +1, rakam ve sembol birlikte +1 (en çok 4). Çubuk renkleri sırasıyla --glint-state-error · --glint-state-warning · --glint-state-info · --glint-state-success.

Karakter sayacı · uyarı ve dolu durumları

Sayaç yalnız sayım yapmaz; maxlength ile birlikte renk durumu da taşır. Uzunluk sınırın %90'ına ulaşınca sayaç --warn (amber), sınıra dayanınca --full (kırmızı + kalın) sınıfına geçer. Aşağıda aynı maxlength değeriyle her iki eşik dondurulmuştur — alana yazıp eşikleri canlı geçebilirsin.

Canlı önizleme
<!-- maxlength → sayaç "n / max" biçiminde; %90'da warn, doluda full -->
<div class="glint-input-group">
  <input class="glint-input" id="title" maxlength="20">
  <label class="glint-label" for="title">Başlık</label>
</div>
Eşikler tamamen oransaldır: len >= max * 0.9 → --warn, len >= max → --full. maxlength tarayıcı tarafından da uygulanır; sayaç hiçbir zaman sınırı aşamaz.

Canlı önizleme

  <label class="glint-label" for="pw">Şifre</label>
</div>
<div class="glint-input-group" data-glint-clearable>

Konumlandırma JS'te _addAction ile yapılır: ikinci ve sonraki butonlar --glint-action-size kadar sola ötelenir, input'un padding-right'ı buton sayısıyla orantılı büyür. Böylece üçten fazla aksiyonda bile çakışma olmaz.

Kombinasyon · şifre toggle + güç + sayaç + temizle

Gerçek bir "yeni parola" alanı genelde tüm yardımcıları birlikte ister. Tek bir gruba data-glint-strength, data-glint-clearable ve maxlength verip type="password" kullandığında Glint hepsini tek seferde kurar: göz ikonu + temizle (×) sağda yan yana dizilir, altta güç çubuğu, sağ-altta canlı sayaç. Hiçbir ekstra JS yok.

Canlı önizleme

  <label class="glint-label" for="newpw">Yeni parola</label>
</div>

04 · Bileşen

Glint Checkbox

Native checkbox üzerine SVG ile çizilen onay işareti animasyonu. Belirsiz (indeterminate) durum, klavye odağı, hover ve koyu mod desteklenir. Native input DOM'da kalır (erişilebilirlik ve form gönderimi korunur).

Temel kullanım

İki kalıp da çalışır: etiketi saran <label> veya for ile ayrı etiket.

Canlı önizleme
<!-- Saran etiket -->
<label class="glint-checkbox-label">
  <input type="checkbox" class="glint-checkbox">
  <span>Şartları kabul ediyorum</span>
</label>

<!-- Ayrı etiket -->
<input type="checkbox" class="glint-checkbox" id="news">
<label for="news">Bülteni al</label>
<label class="glint-checkbox-label">
  <input asp-for="AcceptTerms" type="checkbox" class="glint-checkbox" />
  <span>Şartları kabul ediyorum</span>
</label>
ASP.NET Core'da asp-for bir bool alanı için gizli bir <input type="hidden"> de üretir (işaretsiz değer de gönderilsin diye). Glint yalnızca görünür checkbox'ı sarmalar; bu davranış değişmez.

Belirsiz durum ve "tümünü seç"

Glint Picker

Glint Input'un bir uzantısı. Native HTML5 date / time / datetime-local input'larını sarmalar: native input'u gizler (form değeri ve model binding korunur), yerine Glint stiliyle bir görüntü alanı ve tıklayınca açılan bir popover (takvim ve saat seçici) sunar. Masaüstünde konumlanır, mobilde alttan çıkan bottom-sheet'e dönüşür.

Kullanım normal bir Glint alanıyla aynıdır; tek fark type değeridir. Picker tipi otomatik algılar, ekstra JS gerekmez.

Özellikler

  • Takvim: ay ileri / geri; başlığa tıklayınca bu aya dönüş; önceki / sonraki ay taşma günleri; bugün ve seçili gün işaretleri.
  • Saat seçici: saat ve dakika alanları, yukarı / aşağı spinner (basılı tutunca hızlanır), manuel yazım, ok tuşları; odakta içerik seçilir, blur'da otomatik sıfır eklenir (9 olur 09).
  • Seçim akışı: date'te gün seçince popover kapanır; time ve datetime-local'de footer'daki onay (✓) butonu ile kapanır.
  • Footer: "Bugün", "Temizle" ve (time / datetime'da) "Tamam".
  • Konumlandırma: masaüstünde akıllı yerleşim, scroll / resize'da yeniden hizalanır; mobilde bottom-sheet, arka plan ve kaydırma kilidi.
  • Mobil: alanı seçince native klavye yerine picker açılır.
  • Klavye: alan üzerinde Enter / Space / aşağı ok ile açılır, Esc ile kapanır.
v2 ile gelenler: min / max, devre dışı günler, tarih aralığı (range) seçimi ve grid içi tam ok-tuşu navigasyonu artık destekleniyor. Yeni öznitelikler opsiyoneldir; mevcut basit kullanım aynen çalışır.

Glint, input.glint-color seçicisini type'tan bağımsız sarmalar: native type="color" yerine sıradan bir type="text" alanına da uygulayabilirsin. Native renk diyaloğunun #rrggbb kısıtı yerine, modelinde serbest HEX string tutmak istediğinde (örn. "#3b82f6" bir string property'ye) bunu kullan. Aynı popover, aynı senkron: değer .value üzerinden okunur/yazılır, input/change yayılır.

Canlı önizleme
<!-- type=text: form değeri serbest HEX string ("#3b82f6") -->
<input type="text" class="glint-color" value="#3b82f6" aria-label="Vurgu rengi">
<!-- HEX'i string property'de tutuyorsan metin input'u ideal -->
<input type="text" class="glint-color" asp-for="AccentHex" />
const el = document.querySelector('#cxv-text-input');
Glint.Color.getValue(el);            // -> "#3b82f6"
Glint.Color.setValue(el, '#10b981'); // değeri programatik ata (input/change yayar)
Fark yalnızca taşınan değerin tipi: type="color" tarayıcının renk diyaloğunu da elinde tutar ve değeri her zaman #rrggbb'ye sıkıştırır; type="text" ise düz string saklar (3 haneli #abc dahil esnek HEX okur). Görsel popover ve davranış ikisinde de birebir aynıdır.

Kare swatch varyantı (.glint-color--square)

Tetikleyicideki renk örneği (swatch) varsayılan olarak yuvarlaktır. Native input'a glint-color--square sınıfını eklersen, Glint sarmalayıcıya .glint-color-field--square verir ve swatch köşeleri yumuşatılmış kare (radius 6px) olur — marka/tema rengi alanlarında daha "örnek kutusu" hissi için. Yalnız swatch geometrisi değişir; tüm işlevler (popover, palet, RGB/HSL, kontrast) aynen korunur.

Canlı önizleme
<!-- glint-color--square → kare (radius 6px) swatch -->
<input type="color" class="glint-color glint-color--square"
       value="#8b5cf6" aria-label="Marka rengi">
<input type="color" class="glint-color glint-color--square"
       asp-for="BrandColor" />

Kombinasyon · kare swatch + metin değeri + kalıcı geçmiş

Gerçek dünyada bu üçü çoğu zaman birlikte gelir: bir tema/marka rengi alanı genelde kare örnek kutusu ister, değeri bir HEX string property'de tutulur (type="text") ve kullanıcının son seçtikleri sayfalar arası hatırlansın diye data-recents-key ile localStorage'a yazılır. Bu üç attribute/sınıf çakışmadan birleşir.

Canlı önizleme · birkaç renk seç, sayfayı yenile
<!-- kare swatch + HEX string + kalıcı son-kullanılanlar -->
<input type="text" class="glint-color glint-color--square"
       value="#0ea5e9" data-recents-key="cxv-theme" aria-label="Tema rengi">
<input type="text" class="glint-color glint-color--square"
       asp-for="ThemeHex" data-recents-key="cxv-theme" />
İsimli/gruplu palet · RGB/HSL mod geçişi · defaultMode · WCAG kontrast rozeti özellikleri Glint.Color.config ile gelir ve bu config global, tektir — sayfadaki tüm picker'lar onu paylaşır (per-input config yoktur). Bu yüzden bu davranışlar tek bir merkezî yapılandırmayla yukarıdaki “İsimli palet · HEX/RGB/HSL mod · WCAG kontrast” bölümünde toplu gösterilir; ayrı izole demolar olarak tekrarlanmaz. Varsayılanlar zaten showRgb+showHsl (→ mod toggle) ve showContrast'ı açık getirir; popover'ı açıp HEX altındaki RGB/HSL geçişini ve kontrast rozetlerini bu demoların hepsinde görebilirsin.

Input ↔ Toast Akıllı Hata Gösterimi

Glint Input ve glint-toast bağımsız iki pakettir (ikisi de sıfır bağımlılık). İkisi de yüklüyse gevşek-bağlı işbirliği yapar: sunucudan (FluentValidation) dönen alan hataları, alanın o an görünür olup olmadığına göre akıllıca yönlendirilir. Tek API: Glint.Toast.error(mesaj, alanId).

fieldErrorsDavranış
"smart" (varsayılan)Alan görünürse yalnız alan-içi (yazısız kenarlık + shake); görünmüyorsa özet toast + toast’a tıkla → ilk hatalı alana kaydır+odakla.
"inline"Alan-bağlı hatalar her zaman alan içinde; toast çıkmaz (genel mesajlar yine toast).
"toast"Alan-bağlı hatalar hem inline hem özet toast’ta (tıkla-kaydır aktif).

Varsayılan yazısızdır (sadece kenarlık/shake). Inline mesajı da göstermek için: Glint.configure({ fieldErrorText: true }). Toast tarafı her zaman yazılıdır. Görünür alanların hepsi inline’lanınca toast hiç gösterilmez. Görünmeyen/algılanamayan (visibility:hidden, opacity:0) alan asla sessizce kaybolmaz — fail-safe olarak toast’a düşer.

Canlı önizleme · mod seç, "Sunucu hatası" bas
// Politikayı (opsiyonel) ayarla — varsayılan zaten "smart"
Glint.configure({ fieldErrors: 'smart' });   // 'inline' | 'toast' | 'smart'

// Sunucudan dönen alan hatalarını köprüye ver: alanId görünürse inline,
// değilse özet toast (+ tıkla-kaydır). Tek satır:
Glint.Toast.error('Geçerli bir e-posta girin.', 'Email');
Glint.Toast.error('Bu kullanıcı adı alınmış.', 'UserName');

// Alansız (genel) hata → her zaman toast:
Glint.Toast.error('Kayıt başarısız. Lütfen tekrar deneyin.');
@* Sunucu köprüsü: ModelState hatalarını sayfa init'inde toast motoruna ver.
   glint-toast, window.__glintToasts dizisini açılışta işler (camelCase/PascalCase). *@
<script>
  window.__glintToasts = [
    @foreach (var e in ViewData.ModelState
        .Where(m => m.Value.Errors.Count > 0))
    {
      foreach (var err in e.Value.Errors)
      {
        @:{ type: 1, fieldId: '@e.Key', messages: [{ message: '@err.ErrorMessage' }] },
      }
    }
  ];
</script>
@* type: 1 = ERROR. fieldId = model alan adı (asp-for id ile eşleşir).
   fieldErrors politikası görünür/görünmez kararını otomatik verir. *@
Bu demo için sayfaya glint-toast da yüklüdür. Köprü tamamen opsiyoneldir: toast’ı yüklemezsen Glint.Input.setError tek başına çalışır; input’u yüklemezsen toast tek başına düz bildirim verir.

ASP.NET Core MVC entegrasyonu

Glint, normal HTML üretildiği sürece çalışır; yani Razor tag helper'larıyla (asp-for) tam uyumludur. İki yaklaşım: (1) her alanın yanında satır içi asp-for, (2) tekrarı azaltan, projeye drop-in edilebilen yeniden kullanılabilir yapılar.

1. Doğrulama köprüsü (jQuery unobtrusive)

Glint kendi hata mesajı elemanını (.glint-error-msg) üretir. MVC'nin client-side doğrulamasını buna bağlamak için aşağıdaki köprüyü validation script'lerinden sonra ekle. Picker, native input'u gizlediğinden köprüde ignore: [] şarttır.

// _ValidationScriptsPartial'dan SONRA yükle (jQuery + validate hazır olmalı)
(function () {
  if (!window.jQuery || !jQuery.validator) return;

  function field(el) {
    return window.Glint && Glint.Input ? Glint.Input._getInstance(el) : null;
  }

  jQuery.validator.setDefaults({
    // Picker native input'u hidden yapar; varsayılan ignore:":hidden"
    // bu alanları doğrulama DIŞI bırakır. Hidden alanlar da doğrulansın:
    ignore: [],
    highlight: function (el) {
      var msg = jQuery(el).closest('form')
                 .find('[data-valmsg-for="' + jQuery(el).attr('name') + '"]').text();
      if (field(el)) Glint.Input.setError(el, msg || ' ');
    },
    unhighlight: function (el) {
      if (field(el)) Glint.Input.clearError(el);
    }
  });
})();

Mesajı data-valmsg-for span'inden okumak yaygın bir yoldur; daha sağlam bir yol, doğrulayıcının kendi mesajını showErrors override'ı ile aktarmaktır. Sunucu (ModelState) hataları için sayfa yüklenince geçersiz alanlarda Glint.Input.setError(...)'ı manuel çağır.

2a. Reusable: Tag Helper (önerilen)

En temiz yöntem: kendi <glint-input> tag helper'ın. Tek satırda tam grup markup'ını üretir, asp-for ile model binding ve doğrulama attribute'larını korur.

TagHelpers/GlintInputTagHelper.cs

using Microsoft.AspNetCore.Mvc.Rendering;
using Microsoft.AspNetCore.Mvc.ViewFeatures;
using Microsoft.AspNetCore.Razor.TagHelpers;

namespace YourApp.TagHelpers;

[HtmlTargetElement("glint-input")]
public class GlintInputTagHelper : TagHelper
{
    private readonly IHtmlGenerator _gen;
    public GlintInputTagHelper(IHtmlGenerator gen) => _gen = gen;

    [HtmlAttributeName("asp-for")]
    public ModelExpression For { get; set; } = default!;

    [HtmlAttributeName("label")]
    public string? Label { get; set; }

    // date | time | datetime-local | email | password ... (opsiyonel)
    [HtmlAttributeName("type")]
    public string? InputType { get; set; }

    [HtmlAttributeNotBound, ViewContext]
    public ViewContext ViewContext { get; set; } = default!;

    public override void Process(TagHelperContext context, TagHelperOutput output)
    {
        output.TagName = "div";
        // ZORUNLU: self-closing <glint-input /> kullanımında bile
        // <div>...</div> (içerikle) render edilsin.
        output.TagMode = TagMode.StartTagAndEndTag;
        output.Attributes.SetAttribute("class", "glint-input-group");

        // id'yi açıkça üretiyoruz ki label.for ile birebir eşleşsin
        var id = TagBuilder.CreateSanitizedId(For.Name, "_");
        var text = Label ?? For.Metadata.DisplayName ?? For.Metadata.PropertyName;

        // <input asp-for> eşdeğeri: model binding + data-val-* otomatik
        var input = _gen.GenerateTextBox(ViewContext, For.ModelExplorer,
            For.Name, For.Model, format: null,
            htmlAttributes: new { @class = "glint-input", id });
        if (!string.IsNullOrEmpty(InputType))
            input.Attributes["type"] = InputType;

        var label = new TagBuilder("label");
        label.Attributes["for"] = id;
        label.AddCssClass("glint-label");
        label.InnerHtml.Append(text!);

        output.Content.AppendHtml(input).AppendHtml(label);
    }
}
